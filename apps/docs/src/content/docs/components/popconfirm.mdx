## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `ReactNode` | required | The headline question. |
| `description` | `ReactNode` | — | Helper text below the title. |
| `icon` | `ReactNode` | auto | Default `AlertTriangle` for danger, `Info` otherwise. |
| `tone` | `'default' \| 'danger' \| 'warning'` | `'default'` | Icon colour + OK button variant. |
| `okText` | `string` | `'OK'` | Label for the confirm button. |
| `cancelText` | `string` | `'Cancel'` | Label for the cancel button. |
| `onConfirm` | `() => void \| Promise<void>` | — | Confirm handler. |
| `onCancel` | `() => void` | — | Cancel handler. |
| `confirming` | `boolean` | `false` | Show loading state on the confirm button. |
| `open` / `defaultOpen` / `onOpenChange` | — | — | Standard Radix open state. |
| `side` / `align` | — | — | Standard Radix Popover positioning. |
| `children` | `ReactNode` | required | The trigger element. Should `asChild`-style accept the popover trigger (a single React element). |

## Don't

- Don't use Popconfirm when an immediate Toast undo is more appropriate (e.g. archiving with a 5-second undo window).
- Don't stack multiple Popconfirms inside the same flow — escalate to a Modal.
