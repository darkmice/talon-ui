## Anatomy

```
<Combobox value={v} onValueChange={setV}>
  <ComboboxTrigger size="md">
    <ComboboxValue placeholder="..." />
  </ComboboxTrigger>
  <ComboboxContent>
    <ComboboxInput placeholder="..." />
    <ComboboxList>
      <ComboboxEmpty>Empty state</ComboboxEmpty>
      <ComboboxGroup heading="Section">
        <ComboboxItem value="x">Item</ComboboxItem>
      </ComboboxGroup>
    </ComboboxList>
  </ComboboxContent>
</Combobox>
```

## Props (highlights)

### Combobox
| Prop | Type | Description |
|---|---|---|
| `value` | `string` | Controlled value. |
| `defaultValue` | `string` | Uncontrolled initial value. |
| `onValueChange` | `(value: string) => void` | Commit handler. |
| `open` / `defaultOpen` / `onOpenChange` | — | Popover open state. |

### ComboboxTrigger
Same `size` / `tone` as `SelectTrigger`.

### ComboboxItem
| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | required | Item identifier; also used by cmdk for filtering. |
| `tone` | `'default' \| 'danger'` | `'default'` | Destructive items go red. |
| `disabled` | `boolean` | `false` | Non-selectable. |

## Don't

- Don't use `Combobox` for fewer than ~10 items — a plain `Select` is faster.
- Don't render image content inside `ComboboxItem` unless you also pass a string `value` cmdk can filter by.
