## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | — | Controlled `"HH:MM"` or `"HH:MM:SS"`. |
| `defaultValue` | `string` | — | Uncontrolled initial. |
| `onValueChange` | `(value: string) => void` | — | Commit handler. |
| `withSeconds` | `boolean` | `false` | Adds the seconds column. |
| `hourStep` | `number` | `1` | Hour grid step. |
| `minuteStep` | `number` | `1` | Minute grid step. |
| `secondStep` | `number` | `1` | Second grid step. |
| `placeholder` | `string` | `'HH:MM'` | Shown when no value. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Trigger height. |
| `tone` | `'default' \| 'invalid'` | `'default'` | Border treatment. |

## Don't

- Don't expose seconds to end-users unless your domain requires it (event scheduling, telemetry windows).
- Don't pair a wide step like `minuteStep=30` without a clear caption — users may not realise the granularity.
