## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `id` | `string` | — | Task code (rendered as mono-sm caption). |
| `title` | `ReactNode` | required | Main label. |
| `description` | `ReactNode` | — | Body copy, clamped to 2 lines. |
| `tags` | `Array<{ label, tone? }>` | — | Status pills. |
| `assignees` | `KanbanAssignee[]` | — | Group of assignees; collapses past 3 via `AvatarGroup`. |
| `progress` | `number` | — | 0–100 progress bar. |
| `timestamp` | `ReactNode` | — | Footer caption (e.g. "Updated 2h ago"). |
| `onSelect` | `() => void` | — | When set, card becomes interactive. |
| `selected` | `boolean` | `false` | Highlight with primary ring + border. |
| `size` | `'sm' \| 'md'` | `'md'` | Max-width 280 / 320. |

## Don't

- Don't squeeze more than two tags side-by-side; use vertical stacking.
- Don't use KanbanCard outside a column / board context — orphan cards lose meaning.
