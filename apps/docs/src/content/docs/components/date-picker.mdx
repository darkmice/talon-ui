## Anatomy

```
<DatePicker
  mode="single"
  value={date}
  onValueChange={setDate}
  placeholder="Pick a date"
  size="md"
/>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `mode` | `'single' \| 'range'` | `'single'` | Selection mode. |
| `value` | `Date \| DateRange` | — | Controlled value (shape matches `mode`). |
| `defaultValue` | `Date \| DateRange` | — | Uncontrolled initial value. |
| `onValueChange` | depends on `mode` | — | Commit handler. |
| `placeholder` | `string` | `'Pick a date'` | Shown when no value. |
| `format` | `string` | `'PP'` (single) / `'PP – PP'` (range) | date-fns format string. |
| `disabledDates` | `Matcher \| Matcher[]` | — | react-day-picker matcher(s) for disabled days. |
| `weekStartsOn` | `0..6` | `0` (Sun) | First day of the week. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Trigger height. |
| `tone` | `'default' \| 'invalid'` | `'default'` | Trigger border treatment. |

## Don't

- Don't combine DatePicker with a separate TimePicker in one field — use a single combined picker (planned next).
- Don't drop `placeholder` — empty triggers are visually ambiguous.
