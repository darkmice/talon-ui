## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `number \| string` | — | Controlled value. |
| `defaultValue` | `number \| string` | — | Uncontrolled initial value. |
| `onValueChange` | `(value: number) => void` | — | Convenience callback with the parsed number. |
| `min` | `number` | — | Lower bound. |
| `max` | `number` | — | Upper bound. |
| `step` | `number` | `1` | Increment per arrow / keypress. |
| `precision` | `number` | inferred from `step` | Decimal places to display. |
| `unit` | `ReactNode` | — | Suffix rendered as caption between the input and the arrow column. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Mirrors `Input`. |
| `tone` | `'default' \| 'invalid'` | `'default'` | Mirrors `Input`. |
| `holdStep` | `number` | `100` | Repeat interval (ms) when an arrow is held. |

All non-`size` `<input>` attributes are forwarded.

## Don't

- Don't combine `NumberInput` with a `<Slider>` for the same value — pick one input affordance.
- Don't ship without `min` / `max` if the field has logical bounds. Unbounded numerics invite invalid data.
